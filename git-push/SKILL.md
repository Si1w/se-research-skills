# Git Push Procedure

- Read `git diff && git status` to check for changes.
- Update the documentation in `*.md` files if necessary.
- Use `git add .` to stage all changes.
- Write commit messages with bullet points to summarize the changes made in the commit
- Do not generate coauthor information like "Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>" in commit messages
- Use `git push` to push the changes to the remote repository.
