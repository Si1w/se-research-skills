# Proposed Solution

## Definitive Explanation
- If you introduce a nontrivial novel technical approach, this section must contain the definitive explanation.

## Separation of Concerns
- **Separate from evaluation**: Do not mix in how you intend to evaluate. That belongs in a separate section.
- **Separate from problem description**: Whether the problem is in the introduction or its own section, do not re-describe it here.
- **Separate algorithm from implementation**: Readers want to understand the ideas, algorithm, and overall approach. Implementation details (if needed for replication) must be kept clearly separate.

## Common Failing
- A paper that mixes implementation details with high-level approach description is hard to read and irritates referees. This is a common failing in early drafts.
