# Proposed Solution

## Purpose

If you introduce a nontrivial novel technical approach, you need a section that contains its definitive explanation. Readers are interested in understanding the ideas, the algorithm, and the overall approach.

## Separation of Concerns

- Separate the proposed solution from the way in which you intend to evaluate it, which should come in a separate section.
- Separate the description of the problem (whether it is in the introduction or its own section) from the way you go about solving it.
- Separate implementation aspects from algorithmic aspects. If you evaluated with your own implementation you may need some implementation details, but keep them apart from the description of the algorithm and approach. A paper that mixes implementation details with the high-level approach is hard to read and irritates referees, and it is a common failing in early drafts.
