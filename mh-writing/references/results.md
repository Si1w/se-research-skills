# Results

## Structure
1. **Present evidence first** — results with tables and figures, without judgement.
2. **Then answer RQs** — the authors' view on how the evidence answers each question (which involves judgement).

This separates evidence from interpretation.

## Figure/Table–RQ Mapping
- Aim for a 1:1 correspondence between figures/tables and the research questions they address.
- If it becomes many-to-one, consider refactoring your RQs — perhaps subsidiary questions would help (RQ1.1, RQ1.2, etc.).

## Ideal Paper Structure
- Claims in introduction → map to top-level RQs → each answered conclusively in one place → with reference to a single table/figure → Zeller numbers selected for the abstract.
- Aim for this ideal, then justify any deviations. Do not follow it slavishly if it harms clarity.
